# Windows PC Refactoring Instructions

This document provides Claude Code with instructions for refactoring upstream changes from `aaronsb/claude-code-config` to maintain Windows/PowerShell compatibility.

## When This Applies

These instructions apply when:
1. Pulling changes from upstream (`aaronsb/claude-code-config`)
2. A sync PR has been flagged with "complex changes"
3. New bash scripts have been added without PowerShell equivalents
4. Existing bash scripts have significant logic changes

## Architecture Overview

```
Upstream (aaronsb)          Your Fork (ZZirbel)
─────────────────           ───────────────────
hooks/ways/*.sh      →      hooks/ways/*.sh (keep in sync)
                     →      hooks/ways/win/*.ps1 (PowerShell ports)
settings.json        →      settings.windows.json (template, -Command pattern)
                     →      settings.json (GENERATED by New-ClaudeSettings)
bin/way-match        →      bin/way-match (APE binary, cross-platform)
scripts/             →      scripts/ClaudeConfigManager.psm1 (config manager)
                     →      scripts/Sync-Upstream.ps1 (upstream sync tooling)
```

**settings.json flow:** Upstream `settings.json` uses bash hooks. Your fork maintains `settings.windows.json` with PowerShell hooks. `New-ClaudeSettings` generates the live `settings.json` by expanding `$env:USERPROFILE` to an absolute path.

## Refactoring Process

### Step 1: Identify Changed Scripts

Run this to see what bash scripts changed:
```powershell
git diff upstream/main --name-only -- "*.sh" | Where-Object { $_ -match "hooks/ways" }
```

### Step 2: For Each Changed Script

#### 2.1 Read the bash script changes
```bash
git diff upstream/main -- hooks/ways/{script}.sh
```

#### 2.2 Locate the PowerShell equivalent
The PowerShell equivalent is at: `hooks/ways/win/{script}.ps1`

#### 2.3 Apply the transformation patterns below

### Step 3: Test the changes
```powershell
Invoke-Pester -Path .\tests
```

## Transformation Patterns

### Environment Variables

| Bash | PowerShell |
|------|------------|
| `$HOME` | `$env:USERPROFILE` |
| `$TMPDIR` or `/tmp` | `$env:TEMP` |
| `$PWD` | `$PWD.Path` or `(Get-Location).Path` |
| `${VAR:-default}` | `if ($env:VAR) { $env:VAR } else { "default" }` |
| `export VAR=value` | `$env:VAR = "value"` |

### Path Handling

| Bash | PowerShell |
|------|------------|
| `~/.claude/` | `"$env:USERPROFILE\.claude\"` |
| `/tmp/.claude-*` | `"$env:TEMP\.claude-*"` |
| `dirname "$path"` | `Split-Path -Parent $path` |
| `basename "$path"` | `Split-Path -Leaf $path` |
| `realpath "$path"` | `(Resolve-Path $path).Path` |

### JSON Processing

| Bash (jq) | PowerShell |
|-----------|------------|
| `echo "$json" \| jq -r '.field'` | `($json \| ConvertFrom-Json).field` |
| `jq -r '.field // ""'` | `$obj.field ?? ""` |
| `jq -c '.'` | `$obj \| ConvertTo-Json -Compress` |
| `jq '.field = "value"'` | `$obj.field = "value"` |

### File Operations

| Bash | PowerShell |
|------|------------|
| `cat "$file"` | `Get-Content $file -Raw` |
| `head -n 10 "$file"` | `Get-Content $file -TotalCount 10` |
| `tail -n 10 "$file"` | `Get-Content $file -Tail 10` |
| `touch "$file"` | `New-Item $file -ItemType File -Force` or `(Get-Item $file).LastWriteTime = Get-Date` |
| `rm -f "$file"` | `Remove-Item $file -Force -ErrorAction SilentlyContinue` |
| `rm -rf "$dir"` | `Remove-Item $dir -Recurse -Force -ErrorAction SilentlyContinue` |
| `mkdir -p "$dir"` | `New-Item $dir -ItemType Directory -Force` |
| `[ -f "$file" ]` | `Test-Path $file -PathType Leaf` |
| `[ -d "$dir" ]` | `Test-Path $dir -PathType Container` |
| `[ -x "$file" ]` | `Test-Path $file` (PowerShell doesn't check executable bit) |

### String Operations

| Bash | PowerShell |
|------|------------|
| `${var,,}` (lowercase) | `$var.ToLower()` |
| `${var^^}` (uppercase) | `$var.ToUpper()` |
| `${var/old/new}` | `$var -replace 'old','new'` |
| `${var//old/new}` (global) | `$var -replace 'old','new'` (already global) |
| `${var#prefix}` | `$var -replace '^prefix',''` |
| `${var%suffix}` | `$var -replace 'suffix$',''` |
| `echo "$var" \| tr '/' '-'` | `$var -replace '/','-'` |

### Conditionals

| Bash | PowerShell |
|------|------------|
| `if [ -z "$var" ]; then` | `if ([string]::IsNullOrEmpty($var)) {` |
| `if [ -n "$var" ]; then` | `if (-not [string]::IsNullOrEmpty($var)) {` |
| `if [ "$a" = "$b" ]; then` | `if ($a -eq $b) {` |
| `if [ "$a" != "$b" ]; then` | `if ($a -ne $b) {` |
| `if [[ "$var" =~ regex ]]; then` | `if ($var -match 'regex') {` |
| `$? -eq 0` | `$LASTEXITCODE -eq 0` |

### Loops

| Bash | PowerShell |
|------|------------|
| `for file in *.sh; do` | `foreach ($file in Get-ChildItem *.sh) {` |
| `while read -r line; do` | `foreach ($line in Get-Content $file) {` |
| `find . -name "*.md"` | `Get-ChildItem -Recurse -Filter "*.md"` |

### Process & Commands

| Bash | PowerShell |
|------|------------|
| `command -v git` | `Get-Command git -ErrorAction SilentlyContinue` |
| `which git` | `(Get-Command git).Source` |
| `git ... 2>/dev/null` | `git ... 2>$null` |
| `cmd \|\| true` | `cmd; $null` or `-ErrorAction SilentlyContinue` |
| `cmd && other` | `if ($LASTEXITCODE -eq 0) { other }` |
| `$(cmd)` | `$(cmd)` or `(cmd)` |

### Here Documents

```bash
# Bash
cat <<'EOF'
multi
line
EOF
```

```powershell
# PowerShell
@"
multi
line
"@
```

### Reading YAML Frontmatter

```bash
# Bash - extract field from way.md frontmatter
pattern=$(awk '/^---$/,/^---$/' "$way_file" | grep "^pattern:" | cut -d: -f2- | tr -d ' ')
```

```powershell
# PowerShell - extract field from way.md frontmatter
function Get-WayFrontmatter {
    param([string]$WayFile, [string]$Field)
    $content = Get-Content $WayFile -Raw
    if ($content -match "(?ms)^---\r?\n(.+?)\r?\n---") {
        $yaml = $Matches[1]
        if ($yaml -match "(?m)^${Field}:\s*(.+)$") {
            return $Matches[1].Trim().Trim('"').Trim("'")
        }
    }
    return $null
}

$pattern = Get-WayFrontmatter -WayFile $wayFile -Field "pattern"
```

## Script-Specific Porting Notes

### check-prompt.sh → check-prompt.ps1

Key logic to preserve:
1. Read JSON input from stdin
2. Extract `prompt`, `session_id`, `project_dir`
3. Combine prompt with response topics (from previous Stop hook)
4. Scan all way.md files for pattern/semantic matches
5. Output JSON with `additionalContext`

### check-bash-pre.sh → check-bash-pre.ps1

Key logic to preserve:
1. Extract `command` and `description` from tool input
2. Match against `commands:` patterns in way frontmatter
3. Also check `pattern:` against description

### show-way.sh → show-way.ps1

Key logic to preserve:
1. Check marker file for idempotency
2. Run macro if `macro:` field present
3. Strip YAML frontmatter from content
4. Create marker file
5. Log event

### inject-subagent.sh → inject-subagent.ps1

Key logic to preserve:
1. Read stash from `/tmp/.claude-subagent-stash-{sid}/`
2. FIFO consumption (oldest first)
3. Create teammate marker if applicable
4. Emit ways WITHOUT marker checking (fresh context for subagent)

## New Script Porting Checklist

When a new bash script appears upstream without a PowerShell equivalent:

- [ ] Create `hooks/ways/win/{name}.ps1`
- [ ] Add shebang equivalent: `#Requires -Version 5.1`
- [ ] Convert all bash idioms using patterns above
- [ ] Handle JSON input: `$input = $input | ConvertFrom-Json`
- [ ] Handle JSON output: `$output | ConvertTo-Json -Compress`
- [ ] Use `$env:TEMP` instead of `/tmp`
- [ ] Use `$env:USERPROFILE` instead of `$HOME`
- [ ] Add to `settings.windows.json` if it's a new hook
- [ ] Write Pester test in `tests/{name}.tests.ps1`
- [ ] Run full test suite: `Invoke-Pester -Path .\tests`

After porting, verify:

## Common Pitfalls

### 1. Path Separator Issues
Windows uses `\`, bash uses `/`. The ways system normalizes to `/` in output.
```powershell
# Normalize path separators in output
$wayPath = $wayPath -replace '\\','/'
```

### 2. Temp Directory Location
Windows temp may be `C:\Users\{user}\AppData\Local\Temp` or `C:\Windows\Temp`.
Always use `$env:TEMP` for consistency.

### 3. Exit Codes
Bash `exit 0` maps to PowerShell returning successfully.
Bash `exit 1` maps to `exit 1` or `throw`.
```powershell
# For hook decisions
exit 0  # Allow/continue
exit 1  # Block (for PreToolUse hooks)
```

### 4. JSON Output Encoding
Ensure UTF-8 output for JSON:
```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

### 5. Empty Output
Bash scripts may output nothing on no-match. PowerShell should do the same:
```powershell
if (-not $hasMatch) {
    # Output nothing, just exit
    exit 0
}
```

## Updating settings.windows.json

When upstream adds new hooks to `settings.json`, add the PowerShell equivalent to `settings.windows.json`.

**Important:** Use `-Command` with single-quote wrapping, NOT `-File`. Claude Code's hook runner uses bash (Git Bash on Windows), which eats `$env:USERPROFILE` if passed via `-File`. Single quotes protect the variable so PowerShell can expand it:

```json
{
  "hooks": {
    "NewHookEvent": [
      {
        "matcher": "pattern",
        "hooks": [
          {
            "type": "command",
            "command": "powershell.exe -NoProfile -ExecutionPolicy Bypass -Command '& \"$env:USERPROFILE\\.claude\\hooks\\ways\\win\\new-hook.ps1\"'"
          }
        ]
      }
    ]
  }
}
```

After updating `settings.windows.json`, regenerate `settings.json` with resolved paths:
```powershell
New-ClaudeSettings -Force
```

This reads the template and replaces `$env:USERPROFILE` with the actual absolute path (e.g., `C:/Users/zanzi/.claude/...`), which bash can handle.

## Getting Help

If you encounter a complex bash construct that doesn't have an obvious PowerShell equivalent:

1. Check the existing PowerShell scripts for similar patterns
2. Search for "bash to powershell" conversion guides
3. Ask Claude Code to help with the specific construct
4. Test thoroughly before committing

Remember: The goal is **functional equivalence**, not line-by-line translation. PowerShell may solve the same problem differently, and that's okay as long as the hook behavior matches.
